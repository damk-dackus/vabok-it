A series of tests that needs to be run in succession at rapid speed will best be served by automating the process. Test automation is a very regular practice in testing that does what humans are not able to do by the simple fact that testing is either redundant, excessive, or extreme. In these cases, the tests being performed are not necessarily tests, but are checks put in place to make sure the product being built maintains its composure under stressed conditions. Whether you're implementing new code or ramping a software from a low exposure rate to 100% of users, these conditions promote a need to keep tabs on performance. This is what test automation is for. Take a regression suite, for example. This series of tests ensures all key factors of an application are, in fact, working. Automating a regression suite provides full coverage on everything in that suite.
